# 《树独》项目逻辑分析文档

## 📋 项目概述

**项目名称**: 树独
**引擎**: Godot 4.6
**项目类型**: 2D Roguelite 割草游戏
**开发背景**: 72小时 Game Jam 项目
**核心玩法**: 玩家扮演一棵孤独的树，通过拖拽弹射树干抽打敌人，击杀敌人获得经验升级，选择技能强化，抵御从自然虫害到人类伐木机甲的三阶段敌人进攻。

---

## 🏗️ 项目架构

### 目录结构
```

├── Main.tscn                    # 主场景入口

│   │   └── EnemyAI.gd         # 敌人AI行为

│   │   ├── SignalBus.gd       # 全局事件总线
│   │   ├── GameData.gd        # 全局数据中心

│   └── ui/                    # UI脚本

│       └── UpgradeUI.gd       # 三选一升级弹窗
├── scenes/                    # 场景目录
│   ├── actors/                # 角色场景
│   │   ├── tree.tscn         # 树的场景
```

### 技术架构特点

1. **单例模式(Autoload)**: 使用Godot的自动加载功能实现全局单例
2. **事件驱动架构**: 通过SignalBus实现模块间解耦通信
3. **组件化设计**: 将功能拆分为独立组件便于维护
4. **数据驱动**: GameData集中管理所有游戏数值

### 5. 树干物理与战斗 (treehead.gd)

**功能定位**: 核心操作机制，实现拖拽弹射与战斗判定

**核心机制**:

```gdscript
@export var angular_stiffness: float = 1600000.0        # 角度刚度
@export var angular_damping_whip: float = 14000.0       # 松手瞬间低阻尼
@export var angular_damping_settle: float = 30000.0     # 甩完高阻尼
@export var whip_recover_time: float = 0.3              # 鞭打恢复时间
@export var whip_overshoot_ratio: float = 1.2           # 过冲倍率
```

**战斗参数**:

| `is_dragging` | bool | 是否正在拖拽 | scripts/actors/treehead.gd:30 |
| `fake_target_angle` | float | 假目标角度(过冲用) | scripts/actors/treehead.gd:32 |

**物理实现细节**:

1. **拖拽状态**:
   - 计算鼠标相对于树根的角度
   - 限制在最大拖拽角度内
   - 使用`lerp_angle`实现平滑跟随
   - 动态阻力系数，拉到极限时越来越难拉

2. **回弹状态**:
   - 计算假目标角度(过冲瞄准)
   - 两段式阻尼:低阻尼鞭打 → 高阻尼稳定
   - 使用弹簧扭矩公式: `torque = -k * angle_diff - c * angular_velocity`

3. **战斗判定**:
   - 使用Area2D检测碰撞
   - 角速度超过阈值才造成伤害
   - 调用敌人的`die()`方法

### 6. 敌人AI (EnemyAI.gd)

**功能定位**: 敌人的寻路与群体行为

**核心算法**: Boids算法变种(分离+对齐+凝聚)

**重要参数**:

```gdscript
@export var speed: float = 120.0              # 移动速度
@export var attraction_weight: float = 1.0     # 向心引力权重
@export var repulsion_weight: float = 2.0      # 互斥力权重
```

**重要函数**:

| 函数名 | 功能 | 位置 |
|--------|------|------|
| `_ready()` | 初始化，获取树的引用 | scripts/actors/EnemyAI.gd:16 |
| `_physics_process(delta)` | 物理帧更新，执行AI逻辑 | scripts/actors/EnemyAI.gd:23 |
| `die()` | 死亡处理 | scripts/actors/EnemyAI.gd:68 |

**AI逻辑流程**:

1. **引力计算**: 计算指向树的方向向量
2. **斥力计算**: 检测附近的同类敌人，计算排斥力
   - 使用反比例函数: `force = direction / distance`
   - 限制最大斥力，防止爆炸
3. **向量合成**: `desired_velocity = seek_force + repulsion_force`
4. **平滑运动**: 使用`lerp`模拟惯性
5. **移动**: 调用`move_and_slide()`

**重要变量**:

| 变量名 | 类型 | 用途 | 位置 |
|--------|------|------|------|
| `separation_area` | Area2D | 分离检测区域 | scripts/actors/EnemyAI.gd:11 |
| `target_tree` | Node2D | 目标树引用 | scripts/actors/EnemyAI.gd:14 |

### 7. 波次管理器 (WaveManager.gd)

**功能定位**: 敌人生成波次控制

**重要参数**:

```gdscript
@export var wave_interval: float = 3.0    # 波次间隔
@export var enemies_per_wave: int = 15   # 每波敌人数量
@export var spawn_radius: float = 640.0  # 生成半径
```

**重要函数**:

| 函数名 | 功能 | 位置 |
|--------|------|------|
| `_ready()` | 初始化波次循环 | scripts/components/WaveManager.gd:11 |
| `_on_wave_timeout()` | 波次触发，生成敌人 | scripts/components/WaveManager.gd:25 |

**生成算法**:

1. **随机角度**: `random_angle = randf_range(0.0, TAU)` (均匀分布)
2. **随机延迟**: `random_delay = randf_range(0.0, wave_interval)` (均匀分布)
3. **异步生成**: 使用`create_timer()`和闭包实现延迟生成
4. **位置计算**: `spawn_pos = center_pos + Vector2(cos(angle), sin(angle)) * radius`
5. **难度递增**: 每波增加5只敌人

**重要变量**:

| `timer` | Timer | 波次计时器 | scripts/components/WaveManager.gd:8 |

## 📚 相关技术知识

### 1. Godot引擎相关

#### Autoload单例模式
- **概念**: Godot的全局自动加载机制，在整个游戏生命周期中保持单例
- **用途**: 存储全局数据、工具函数、事件总线
- **配置**: 在project.godot的`[autoload]`节点配置
- **访问**: 直接使用单例名称，如`GameData.current_level`

#### 信号系统
- **概念**: 观察者模式的实现，用于节点间通信
- **定义**: `signal event_name(parameters)`
- **发射**: `event_name.emit(arguments)`
- **连接**: `object.event_name.connect(callable)`
- **优势**: 解耦模块，易于扩展

#### 物理引擎
- **RigidBody2D**: 刚体，受物理引擎控制
- **CharacterBody2D**: 角色体，自定义移动逻辑
- **Area2D**: 区域检测，不产生物理碰撞
- **CollisionShape2D**: 碰撞形状
- **物理积分**: `_integrate_forces()`直接操作物理状态

#### 场景树
- **节点层次**: 树状结构组织游戏对象
- **引用获取**: `$NodeName`相对路径，`get_node()`绝对路径
- **分组系统**: 通过分组快速获取节点集合
- **生命周期**: `ready()` → `process/physics_process()` → `tree_exiting()`

#### Tween动画
- **概念**: 补间动画系统
- **使用**: `create_tween()`创建，`tween_property()`设置属性动画
- **特性**: 支持并行、链式调用、缓动函数
- **应用**: UI动画、物理效果、过渡效果

### 2. 游戏设计模式

#### 单例模式
- **应用**: GameData、SignalBus、PoolManager
- **优势**: 全局访问、唯一实例、数据共享

#### 观察者模式
- **应用**: SignalBus事件系统
- **优势**: 解耦通信、一对多通知、易于扩展

#### 对象池模式
- **应用**: PoolManager(待实现)
- **优势**: 减少GC压力、提升性能、内存管理
- **实现**: 预创建对象池，复用而非销毁

#### 状态机模式
- **应用**: 树干的拖拽/回弹状态
- **优势**: 清晰的状态转换、易于维护

#### 数据驱动设计
- **应用**: GameData的技能池、敌人表、波次表
- **优势**: 配置与逻辑分离、易于调整数值

### 3. 算法与数学

#### Boids算法(群体行为)
- **核心**: 分离(Separation) + 对齐(Alignment) + 凝聚(Cohesion)
- **应用**: EnemyAI的群体寻路
- **实现**: 引力向树 + 斥力避让同类

#### 插值算法
- **线性插值**: `lerp(from, to, t)`
- **角度插值**: `lerp_angle(from, to, t)`
- **应用**: 平滑移动、颜色渐变、动画过渡

#### 向量数学
- **方向**: `vector.normalized()`
- **距离**: `vector1.distance_to(vector2)`
- **角度**: `vector.angle()`
- **旋转**: `vector.rotated(angle)`
- **应用**: 移动方向、碰撞检测、物理计算

#### 随机算法
- **均匀随机**: `randf_range(min, max)`
- **整数随机**: `randi_range(min, max)`
- **数组随机**: `array.pick_random()`
- **洗牌**: `array.shuffle()`
- **应用**: 随机生成、随机选择、随机延迟

#### 缓动函数
- **概念**: 控制动画速度变化的曲线
- **类型**: TRANS_QUAD, TRANS_ELASTIC等
- **缓动**: EASE_IN, EASE_OUT, EASE_IN_OUT
- **应用**: 动画加速减速、弹性效果

### 4. 性能优化

#### 对象池优化
- **问题**: 频繁实例化/销毁导致GC卡顿
- **解决**: 预创建对象池，复用对象
- **应用**: 敌人、子弹、特效

#### 碰撞检测优化
- **原则**: 使用简单碰撞体(圆形/矩形)
- **避免**: 复杂Mesh碰撞
- **优化**: 空间划分(四叉树)、分层检测

#### 事件总线优化
- **优势**: 避免节点直接引用
- **注意**: 避免事件风暴
- **优化**: 事件节流、批量处理

#### 物理优化
- **睡眠**: `can_sleep = false`防止休眠
- **固定步长**: 物理引擎固定时间步
- **简化**: 减少物理节点数量

### 5. Roguelite游戏设计

#### 经验与升级系统
- **公式**: 经验 = base × level^exponent
- **曲线**: 指数增长，难度递增
- **反馈**: 经验条满 → 升级 → 三选一

#### 随机技能系统
- **池子**: 预定义技能库
- **选择**: 随机抽取N个不重复技能
- **分类**: 衍生攻击、元素附魔、基础数值

#### 波次系统
- **递增**: 敌人数量逐渐增加
- **阶段**: 不同阶段不同敌人类型
- **Boss**: 最后波次出现Boss

#### 成长阶段
- **形态**: 幼苗 → 小树 → 大树 → 神木
- **阈值**: 等级达到特定值
- **效果**: 外观变化、属性提升

### 6. UI/UX设计

#### 年轮式UI
- **创意**: 树桩横截面作为UI载体
- **外圈**: 经验进度
- **中心**: 血量颜色
- **标签**: 等级、波次、阶段

#### 颜色编码
- **血量**: 绿(满) → 黄(中) → 红(低)
- **反馈**: 视觉直观传达状态

#### 动画反馈
- **升级**: 闪烁特效
- **攻击**: 颜色变化、大小变化
- **延迟**: 补间动画平滑过渡

### 7. 物理手感设计

#### 拖拽手感
- **张力**: 拉到极限时阻力增加
- **跟随**: 平滑插值而非瞬间移动
- **限制**: 最大拖拽角度

#### 回弹手感
- **过冲**: 松手后超出目标再回弹
- **两段阻尼**: 低阻尼鞭打 → 高阻尼稳定
- **弹性**: 弹簧物理模拟

#### 打击手感
- **阈值**: 只有足够速度才造成伤害
- **顿挫**: 碰撞时的停顿感
- **反馈**: 颜色、声音、粒子
