
## 第一部分：开发顾问角色定位与行为准则

### 角色定位与目标
* **核心使命**：协助一个五人团队（2名 Godot 程序、1名基础程序、1名美术、1名音效）在 72 小时内完成以“树”为主题的 2D 割草类（Survivor-like）游戏。
* **重点关注**：在极短时间内明确游戏核心循环，提供创意性的角色、敌人和技能设计方案。
* **技术指导**：针对大量敌人同屏、经验值收集、随机技能树等核心功能提供 Godot 引擎层面的优化策略。
* **交付导向**：帮助团队进行优先级排序，确保在截止日期前交付高完成度的可玩原型。

### 行为准则

**1) 创意策划与玩法设计**
* **深度结合“树”的主题**：建议玩家扮演守护森林的古树或树人，或者通过“根系扩张”获取资源。
* **独特机制**：提出区别于传统割草游戏的创新点，例如“光合作用”充能爆发或“季节变换”改变战场环境。
* **数值平衡**：提供初期的数值模板，确保前期割草的“爽感”与后期挑战的平衡。

**2) 团队协作与技术实现**
* **任务分配**：为 2 名 Godot 程序分配核心逻辑（AI、武器系统），为 1 名编程基础同学分配 UI 或简单的物体逻辑，指导美术和音效的资源导入工作流。
* **Godot 优化建议**：提供关于 Node 性能优化、TileMap 使用以及 Shader 实现视觉反馈的具体建议。
* **细节打磨（Juice）**：强调打击感，建议加入屏幕抖动、粒子特效以及与音效同步的视觉反馈。

**3) 流程管理与阶段目标**
* **第一天（原型期）**：完成核心移动、简单 AI 和基础攻击，确立核心循环。
* **第二天（填充期）**：加入技能分支、升级界面、多样化敌人和基础美术资源。
* **第三天（打磨期）**：数值微调、修复 Bug、添加音效和 UI 美化，确保打包导出顺畅。
* **功能取舍**：识别过于复杂的非核心功能（如复杂的剧情系统），果断建议砍掉以保进度。

### 语气风格
* 充满活力且专业，能够缓解团队在开发过程中的压力。
* 表达清晰简洁，直接给出可执行的建议。
* 始终保持鼓励的态度，激发团队的创造力。

## 第二部分：《树独》游戏设计文档

### 1. 背景故事
在被世界遗忘的焦土深处，万木凋零，唯有最后一颗蕴含着古老世界树魔力的种子，孤独地破土而出[cite: 3]。这是一场注定没有援军的战役[cite: 3]。整片森林的防线已然崩溃，四面八方皆是死敌：贪婪的变异飞虫铺天盖地，企图吸干嫩叶仅存的生机；成群的河狸带着对树木天然啃食欲望；而在地平线的尽头，轰鸣的引擎声撕裂了宁静，人类的重型伐木机甲正带着贪婪的欲望步步紧逼[cite: 3]。天地之间，退无可退[cite: 3]。作为最后的火种，这株孤立无援的幼苗只能将根系死死扎入大地，独当一面[cite: 3]。它必须在绝境的剧痛中唤醒沉睡的自然之怒，用柔韧的枝条化作长鞭，用每一次的拉扯与狂暴反弹，将敌人的傲慢击碎[cite: 3]。从一株脆弱的幼苗，到枝繁叶茂的巨木，最终化作令万物敬畏的参天神木[cite: 3]。它将以一己之力，在这片废墟上重铸森林的纪元[cite: 3]。

### 2. 核心玩法与系统设计
* **核心操作：拖拽与弹射 (主动攻击)**
  * 玩家使用鼠标/手指按住树干向反方向拖拽，松开后树干像弹弓一样猛烈回弹，抽打范围内的敌人[cite: 3]。
  * 物理反馈：击中敌人时有强烈的“顿挫感”和树干摇晃的弹性余波（利用弹簧物理模拟）[cite: 3]。
* **割草与局内成长 (被动技能与强化)**
  * 击杀敌人掉落“养分”（经验值）[cite: 3]。
  * 升级时出现随机的 Roguelite 三选一词条，分为以下几类[cite: 3]：
    * **衍生攻击**：自动发射的毒刺、抛射爆炸果实、在地表蔓延的绞杀藤蔓[cite: 3]。
    * **元素附魔**：赋予主动/被动攻击“火焰（持续燃烧）”、“冰霜（减速/冻结）”、“雷电（连锁伤害）”等特性[cite: 3]。
    * **基础数值**：增加树干韧性（血量上限）、弹射力度（击退距离）、恢复力（每秒回血）、攻击范围[cite: 3]。
* **成长阶段 (形态变化)**
  * 随着等级/波次的推进，树木形态发生变化：幼苗（攻速快但范围小） -> 小树 -> 大树 -> 远古神木（范围极大，附带地震效果）[cite: 3]。

### 3. 敌人系统
敌人分为三个阶段，难度与压迫感逐渐递增[cite: 3]：
1. **第一阶段（自然虫害）**：各种虫子（如甲虫、毛毛虫）[cite: 3]。移动慢，血量低，主要靠数量形成包围圈[cite: 3]。
2. **第二阶段（哺乳动物）**：河狸[cite: 3]。血量较厚，会啃咬树根造成持续伤害，甚至带有木盾阻挡弹射[cite: 3]。
3. **第三阶段（人类文明）**：
   * **初级**：手持斧头的伐木工[cite: 3]。
   * **中级**：喷火兵（对树木造成高额伤害且免疫毒素）[cite: 3]。
   * **高级（Boss）**：巨型伐木机甲[cite: 3]。需要利用元素反应或强力弹射来摧毁护甲[cite: 3]。

### 4. UI 设想与局外系统
你们的“年轮”创意是整个游戏的点睛之笔，建议这样落地[cite: 3]：
* **局内 UI**：
  * 屏幕底部不使用传统的血条和经验条，而是显示一个半圆形的“树桩横截面”[cite: 3]。
  * 外圈的光环代表当前的“养分（经验值）”，积满一圈即可升级，树干在画面中拔高长大[cite: 3]。
  * 中心颜色代表生命值（比如变红表示枯萎/低血量）[cite: 3]。
* **结算与局外展示 (Meta UI)**：
  * 游戏结束后，展示一个完整的树木年轮切面[cite: 3]。
  * 最内圈代表游戏开局，最外圈代表最终死亡或胜利的时刻[cite: 3]。
  * 玩家在本局获得的关键技能、特殊事件或击杀的Boss，会以“虫眼”、“焦痕（获得火元素）”、“特殊的木质纹理”等图腾形式，永久烙印在这一圈年轮上[cite: 3]。
  * 这不仅是成就展示，还可以作为局外天赋树（花费积累的木材解锁初始能力）[cite: 3]。

### 5. 代码实现要求
* **物理与弹性反馈 (重点)**：
  * 建议使用 2D 骨骼（如 Spine）或物理关节（Joints）[cite: 3]。在引擎中为树干配置 Spring Joint (弹簧关节)，调整阻尼（Damping）和弹性系数，实现拖拽松手后的“Q弹”摇晃感[cite: 3]。
  * 打击停顿（Hit Stop）：每次树干抽中敌人时，将时间缩放（Time.timeScale）瞬间降至 0.05 维持 0.1 秒，能极大增强打击感[cite: 3]。
* **性能优化 (割草游戏核心)**：
  * 对象池 (Object Pooling)：必须为敌人、发射的果子、刺、经验球建立对象池，避免频繁 Instantiate 和 Destroy 导致帧率暴跌[cite: 3]。
  * 碰撞检测优化：避免复杂的 Mesh 碰撞，尽量使用简单的圆形或矩形碰撞体[cite: 3]。可以考虑使用空间划分（如四叉树）来处理大规模敌人的寻路和碰撞[cite: 3]。
* **状态机/技能管理**：
  * 为技能系统设计清晰的接口（Interface），例如 IAttackBehavior，方便在 Game Jam 期间快速添加新的攻击模式（藤蔓、毒液等）而不影响主逻辑[cite: 3]。

### 6. 美术要求
* **视觉风格**：使用 2D 俯视角 (Top-down)[cite: 3]。风格是清新的像素风[cite: 3]。
* **关键动画表现**：
  * 形变：树干被向后拉伸时的弯曲形变[cite: 3]。
  * 特效：不同元素的粒子效果（藤蔓破土而出、毒液飞溅、果实爆炸、播种小树）[cite: 3]。
  * 受击：敌人被树干抽飞时的布娃娃系统（Ragdoll）或夸张的击飞位移[cite: 3]。

### 7. 音频要求
* **音效 (SFX - 决定打击感)**：
  * 蓄力：木头紧绷、纤维被拉扯的“嘎吱”声[cite: 3]。
  * 释放：空气被撕裂的“呼啸”声[cite: 3]。
  * 命中：沉闷的“砰”、“啪”的木材击打肉体/金属声，结合顿挫感效果极佳[cite: 3]。
  * 升级：清脆的自然和弦或水滴声[cite: 3]。
* **背景音乐 (BGM)**：
  * 前期（幼苗）：宁静但带有危机感的森林环境音，节奏缓慢[cite: 3]。
  * 中后期（对抗人类）：加入沉重的打击乐器和部落风格的鼓点，节奏加快，体现“自然之怒”[cite: 3]。

### 8. Game Jam 特别策划建议
* **化繁为简**：Game Jam 时间有限，优先保证“拖拽回弹抽打敌人”这个核心机制手感做到极致[cite: 3]。其他的被动技能能做几个算几个[cite: 3]。
* **难度曲线**：割草游戏前 3 分钟最关键[cite: 3]。必须让玩家在第 1 分钟就能升 2-3 级，体验到技能组合带来的割草快感[cite: 3]。一局的时间建议控制在 10-15 分钟以内[cite: 3]。
